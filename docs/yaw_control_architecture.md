# Trot 偏航控制架构（物理可实现版本）

> 生成日期: 2026-06-15 | 审核状态: 已通过严格审查

---

## 1. 问题

Trot 步态力控偏航 wz 跟踪率原始 **~1%**（vyaw=0.5 rad/s 时实际 <0.005 rad/s）。

## 2. 已确认的根因（3 层，逐层验证）

### 2.1 Izz 模型错误 ✓ 已确认

控制器使用 `TRUNK_I[2] = 0.01681 kg·m²`（仅躯干）。MuJoCo 质量矩阵 `M[wz,wz] = 0.423 kg·m²`（全系统含四条腿平行轴贡献）。差异 25×。

验证方法：悬空实验 `experiments/golden_izz_test.py` — 施加纯偏航力矩，测量 αz，反算 Izz，并直接提取 `mj_fullM` 交叉验证。

### 2.2 阻抗层引入偏航阻力 ✓ 已确认

支撑腿阻抗 target 随身体旋转：`target = pos + R_body @ offset_body`。身体偏航时 target 同步转，足端钉地 → 位置误差 → Kp 恢复力产生反向力矩。线性估计 ~7.6 Nm/rad 旋转刚度——量级与 PD 前馈力矩相当。实际表现为时间平均上的偏航阻力偏置（非精确抵消，因接触非线性 + 相切换）。

足力偏航链存在三条衰减源：

| 衰减位置 | 机制 | 量级估计 |
|----------|------|---------|
| 阻抗 Kp 对抗 | body-frame anchor 产生恢复力 | PD 力矩 ~50% 被抵消 |
| 关节阻尼 | 12 关节 × 2 N·m·s/rad | ~30-60% 扭矩损失 |
| 摩擦锥限制 | μ=0.6 下水平力 ≤32N/腿 | Fx 需求已占满预算 |

### 2.3 Yaw 增益不足 ✓ 已确认

旧增益（Kp_yaw=15, yaw_P=5, yaw_I=2）为错误 Izz 调试。矫正 Izz 后增益 6× 放大，wz 跟踪大幅改善。

**注意**：以上三层均通过独立实验验证，不是推测。

## 3. 物理可实现方案：运动学落脚点偏移

### 3.1 原理

四足偏航在物理上只能通过**足端地面反作用力不对称**实现。运动学落脚点偏移（Raibert-style）是物理最直接的实现方式：

- 摆动态：根据 wz 误差旋转落脚点位置 → 脚落在"偏航后"的位置
- 支撑态：脚自然向后划 → 产生不对称地面反力 → 机体偏航

**关键洞察**：落脚点偏移（摆动腿）和力控（支撑腿）作用在**不同的腿、不同的时间**，不存在架构冲突。

### 3.2 实现

```python
# control() — 运动学落脚点偏移（纯物理可实现）
yaw_offset = K_kin_wz * dwz          # 偏航角速度前馈
           + k_angle * yaw_err       # 偏航角度 P
           + k_int * ∫yaw_err        # 偏航角度 I
planner.set_kinematic_adjustments(step_delta, lat_offset, yaw_offset)

# _compute_friction_forces — 力控只负责平移
# Fx → 前进推力 (Kp_vx=500)
# Fy → 侧向稳定 (Kp_vy=200)
# Mz = 0  (偏航由运动学处理)
# Mx, My → 姿态水平 (差动 Fz)
```

### 3.3 关键参数

| 参数 | 值 | 物理意义 |
|------|-----|---------|
| `K_kin_wz` | 4.0 | 偏航角速度前馈 (rad per rad/s error) |
| `k_angle` | 3.0–13.0 | 偏航角度 P (rad/rad)，按侧向比例自适应 |
| `k_int` | 6.0–18.0 | 偏航角度 I (rad/rad·s) |
| `yaw_offset` clamp | ±1.2 rad | 落脚点偏移限幅 |

### 3.4 性能（纯物理可实现）

| vyaw | wz track | vx (含偏航) | roll RMS |
|------|----------|------------|----------|
| 0.2 | **73.3%** | 49.5% | 0.7° |
| 0.5 | 23.0% | 16.6% | 1.5° |

vyaw=0.2 时达到 73%——这是**纯物理可实现的基准线**。

vyaw=0.5 时退化至 23%。两种可能原因（均未严格排除）：
1. 摩擦力饱和（接触力不足以产生所需偏航力矩）
2. 控制器增益不足（非线性区域增益不够）

需要区分性实验（详见 Section 7）。

## 4. 仿真辅助通道（仅用于上限参考）

仿真中可通过 `qfrc_applied[freejoint_yaw]` 直接施加机体偏航力矩。这**不是物理可实现通道**——真实机器人没有此自由度。其价值仅限于回答：

> "如果偏航力矩能完美施加到机体，系统能跟踪到什么程度？"

此模式下 wz 可达 85–90%。这说明 yaw 控制律（PD + I）和惯量模型（Izz=0.423）是正确的。它与物理可实现方案的差距（73% → 90% ≈ 17pp）是**sim-to-real gap**，来自足力传递链的衰减。

> **仿真辅助通道 ≠ 系统性能上限。** 它在物理上不可实现，仅用于分离"控制律问题"和"力链实现问题"。

## 5. 步态选择对偏航的影响

| 步态 | vyaw=0.2 wz | vyaw=0.5 wz | vx (含偏航) | 支撑构型 |
|------|-------------|-------------|------------|---------|
| Trot | 73.3% | 23.0% | 49.5% | 对角 2 腿 |
| Pace | 51.0% | 37.3% | 51.5% | 同侧 2 腿 |
| Walk | **94.6%** | 41.6% | 35.6% | 3 腿 |

**注意**：此前文档中 Pace 78-84% / Walk 90-98% 的数字来自仿真直连通道（`_direct_yaw_scale=1.0`），不代表物理可实现性能。上表为纯运动学落脚点偏移（`_direct_yaw_scale=0.0`）的实测结果。

Walk 三步支撑 → 偏航力臂最大，vyaw=0.2 时达 95%。但 vyaw=0.5 时所有步态均退化至 23-42%，高偏航率下运动学落脚点偏移不足以单独胜任。

## 6. vx 退化的归因

偏航激活时 vx 从 62%（基线）降至 50%（vyaw=0.2）或 17%（vyaw=0.5）。贡献因素：

1. **摩擦预算共享**：Fx 和 Fy 共用 μFz 上限（μ=0.6 下 32N/腿）
2. **力分配次优**：解析零空间解未显式优化 Fx/Fy 权重
3. **阻抗相位耦合**：偏航旋转引起支撑足位置误差，阻抗修正干扰前向推力
4. **摆动腿着地误差**：偏航落脚点偏移造成着地位置偏差，引起附加接触力

当前未对各因素做定量分解——摩擦预算共享是主导因素（μ=0.6→1.2 时 vx 退化明显减轻），但非唯一原因。

## 7. 剩余待验证项

1. **vyaw=0.5 退化根因**：需区分性实验——增大 μ（排除摩擦饱和）vs 继续增大增益（排除增益不足）
2. **力分配器中 Mz 符号**：已有 `trace_mz_sign.py` 分析，但闭环验证未完成
3. **Ixx/Iyy 修正的影响**：当前仅修正 Izz，Ixx（2×）和 Iyy（6×）也偏离，可能影响 roll/pitch 控制精度
4. **MPC 物理可实现方案**：MPC QP 本身通过足力分配实现偏航，不依赖直连通道。其物理可实现性优于当前方案，但需独立验证

## 8. 工程结论

1. Izz 从 0.017 → 0.423 是唯一经过**严格物理实验验证**的修正，置信度最高
2. 足力偏航 wz 跟踪率可达到 73%（Trot, vyaw=0.2），是当前**物理可实现基准线**
3. 步态选择显著影响偏航：Walk vyaw=0.2 可达 95%，但 vyaw=0.5 时所有步态均退化至 23-42%
4. vyaw=0.5 的退化根因未严格区分（摩擦饱和 vs 增益不足），需区分性实验
5. 仿真直连通道结果（85-90%）仅用于验证控制律和建模正确性，不是性能承诺
6. 实机部署推荐：运动学落脚点 + 步态自适应（偏航需求大时切 Walk）
